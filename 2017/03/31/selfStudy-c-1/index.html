头文件(包含文件)

C语言的传统是，头文件使用扩展名h，例如 math.h ，C++中这些老式的C的头文件保留了扩展名，即C++仍可以使用这种文件。而C++的头文件没有扩展名，有些C头文件被转换为C++头文件，这些文件被重新命名，具体做法是：去掉扩展名h，名称前边加上前缀c,表明来自C语言，例如 cmath 。对于纯粹的C++头文件来说，去掉扩展名不只是形式上的变化，没有h的头文件可以包含名称空间，用一个表格做总结：

头文件类型 约定 示例 说明
C++旧式风格 以.h结尾 iostream.h C++可以使用
C旧式风格 以.h结尾 math.h C、C++可以使用
C++新式风格 没有扩展名 iostream C++可以使用，使用namespace std
转换后的C 加上前缀c，没有扩展名 cmath C++程序可以使用，可以使用不是C的特性，如namespace std

cout && cin

来看一下很很很简单的代码：

cout是一个预定于的对象，<<指出了信息的流动路径，即把这个字符串或者整数发送给cout,cout很智能，知道如何显示字符串、数字和单个字符。cout的智能行为源自于C++的面向对象特性，C++插入运算符（<<）会根据其后的数据类型相应的调整其行为，这是一个运算符重载的例子(后面再说吧)。endl是一个特殊的C++符号，表示重起一行，在输出流中插入endl将导致屏幕光标移到下一行的开头，endl位于std名称空间中，这类字符称为控制字符。还有一种换行的方法使使用C语言的符号’\n’。

endl可以确保程序继续运行前刷新输出，而‘\n’不能提供这样的保证，因为实际上‘\n’是一种被称为“转义序列”的按键组合。
题外话：你能实现一个算法，输入整数类型，输出对应的字符串吗？

对于cin，上述语句中信息从cin流向变量a，iostream文件将cin定义为一个表示这种流的对象。输出时，<<运算符将字符串插入到输出流中，输入时，>>运算符从输入流中抽取字符。

类与函数

不多讲，C++提供了两种发送消息的方式：

  • 使用类方法。(本质就是函数调用)
  • 重新定义运算符。(cin和cout采用的就是这种方式)

C++中函数不能嵌套，使用时需要先声明。
