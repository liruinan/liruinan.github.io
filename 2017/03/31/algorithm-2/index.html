分治法

发表于 2017-03-31 | 分类于 算法的奥妙

闲聊

什么是分治？查了一些资料，就是分而治之！其实也是废话，维基上说：分治法是建基于多项分支递归的一种很重要的算法范式。就是把一个复杂的问题分成两个或更多的相同或相似的子问题，这些子问题相互独立且与原问题性质相同，直到最后子问题可以简单的直接求解，原问题的解即子问题的解的合并。很多高效的算法都是运用分治的技巧！例如：递归、快排、傅里叶变换。想运用好分治法，需要大量的实践，才能做到手到擒来。
分治法解题的步骤：

  • 分解：将我们要解决的问题分解成若干个相对独立、与原问题解法相同的同类问题
  • 求解：如果子问题已分解到足够小并且我们可以解决的时候，直接去解，否则的话继续递归
  • 合并：根据原问题的要求，对各个子问题以一定的形式进行合并

简单且经典的例子是避免枯燥的最好方式，这里我举个简单的归并排序的例子，正好也复习了一遍归并排序算法。哈哈哈😆，我们以经典的2-路归并排序为例来讲，首先形象的看一下排序过程:

上面的图我就不解释了，说一下归并排序原理：假设初始序列有n个记录，开始可看成n个有序的子序列，正如上面所看到的，49、38、65这些数字分别可看成是一个个的子序列，每个子序列的长度为1，然后两两归并，会得到⎡n/2⎤个长度为2或者1的子序列，然后两两归并，⋅⋅⋅⋅，直到得到长度为n的有序序列为止，也就是最后一趟了。归并排序的核心操作是：将前后相邻的有序序列归并为一个有序序列,核心算法如下：

  • 分解：将含有n个元素的待排序表分成两个子表，对子表进行递归的分解

归并的性能分析

空间效率：在Merge()操作中，辅助空间需要占用n个单元，所以归并排序的空间复杂度为O(n)。
时间效率：每一趟的归并的时间复杂度为O(n)，一共需要进行⎡logn⎤趟归并，所以算法的时间复杂度为O(nlogn)
稳定性：小跨度，稳定！

对于N个元素进行k-路归并排序时，排序的趟数m满足k^m=N。
